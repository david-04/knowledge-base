Event types


Direct events

  • are handled only by one element


Bubbeling routed events

  • travel up in the hierarchy from the child to its parents
  • any element in the hierarchy can mark the event as handled and prevent further propagation


Tunneling routed events

  • the name is prefixed with Preview (e.g. PreviewClick)
  • tunnelling events travel down the hierarchy from the root element to the children
  • any element in the hierarchy can handle the event and prevent further propagation
  • parent elements can prevent events from reaching their children



Registering event handlers


<DockPanel   Keyboard.PreviewKeyDown="OnKeyDownPreview">      <!-- intercept the KeyDown event during the tunneling phase -->

    <TextBox Keyboard.KeyDown="OnKeyDown"/>                   <!-- process the KeyDown event during the bubbeling phase   -->



Handling events


private void OnKeyDownPreview(object sender, KeyEventArgs e)  // handle the tunneling event

{

    e.Handled = false;

}


private void OnKeyDown(object sender, KeyEventArgs e)         // handle the bubbeling event

{

    e.Handled = true;

}
