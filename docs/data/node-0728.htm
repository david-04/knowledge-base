Help


> git help command                                  // get help for a particular command

> git command --help

> man git-command



Snapshots


> git archive myBranch --prefix="myProject/"        // create a gzipped snapshot (workspace) of the latest commit in myBranch

        | gzip > `git describe master`.tar.gz


> git archive master --prefix="myProject/"          // same as above but in ZIP format

        --format=zip > `git describe master`.zip



References


734713bc047d87bf7eac9674765ae793478c50d3            // the SHA-1 hash of the commit (it's enough to provide only the first letters)

myBranch                                            // refers to the last commit of the specified branch

HEAD^                                               // the commit before the current head (i.e. the next-to-last commit)

HEAD^^                                              // the parent of the parent

d921970^2                                           // the second-last parent (i.e. the branch that was merged into the current one)

HEAD~3                                              // the third-last parent (can pass any number)


> git rev-parse myBranch                            // returns the SHA-1 of the latest commit on myBranch


master..myBranch                                    // all commits on myBranch that occurred after diverting from master

myBranch --not master                               // same as above

^master myBranch                                    // same as above

myBranch..master                                    // all new master commits that are not (yet) part of myBranch

origin/master..HEAD                                 // all that's about to be pushed (HEAD is optional and can be omitted)

myBranch1 myBranch2 ^master                         // everything in myBranch1 and myBranch2 that is not in the master branch

master...myBranch                                   // all commits that are either in master or myBranch but not in both



Submodules


> git submodule add https://...                     // project the specified repository as a subfolder of the current repository

                                                    // (this subfolder is not version-controlled by the wrapping parent repository)

> git submodule init                                // initialise the submodule(s)

> git submodule update                              // clone/pull the submodule(s)
