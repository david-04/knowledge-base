Staging changes


> git add my-file.txt                               // stage a single file

> git add MyFolder                                  // recursively stage all files in the given folder


> git rm          my-file.txt                       // remove the file locally and stage a delete command

> git rm -f       my-file.txt                       // remove the file even if it has pending (staged or unstaged) changes

> git rm --cached my-file.txt                       // stage the file as deleted but preserve the local file as untracked item


> git mv original-file.txt renamed-file.txt         // rename a file (works like rm+add, does not migrate the version history)



Commiting changes


> git commit my-file.txt                            // commit all staged changes (or the selected file[s] only)

> git commit -m "comment"                           // pass the commit comment as command line paramter

> git commit -v                                     // when prompting for the commit comment: show the actual file diff

> git commit -a                                     // do an implicit "git add" (commit changes even if they have not been staged yet)

> git commit --amend                                // merge the current changes into the previous commit (altering it)



Reverting changes


> git reset        HEAD   my-file.txt               // unstage a previously staged (but not yet committed) file

> git reset --hard HEAD                             // revert the staging area and working directory to the last commit

> git reset ...   8c9zs7                            // revert to a specific commit


> git reset --soft  HEAD~                           // move the HEAD tag in the repository back by one commit

> git reset --mixed HEAD~                           // like --soft, but also affects the staging area (--mixed can be omitted)

> git reset --hard  HEAD~                           // like --mixed, but also affects the working directory


> git reset HEAD~1                                  // remove the last commit from the repository and staging area but preserve the working directory copy

> git reset 8c9zs7                                  // like above, but remove everything after the specified commit


> git checkout -- my-file.txt                       // revert local changes and replace the file with the commited (or staged?) version



Remvoing untracked files


git clean -d -f                                     // delete all untracked files and folders (but not those matched by .gitignore)

git clean -d -f -x                                  // delete even those untracked files coverd by .gitingore

git clean -d       -n                               // dry-run, only show the anticipated cleanups

git clean          -i                               // interactive mode
