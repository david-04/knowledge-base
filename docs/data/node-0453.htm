Online status


navigator.onLine                                    // HTML5: check if an online connection exists

window.on{on|off}line                               // HTML5: fired when the network connection is established/lost



Defining a cache manifest


#Comment                                            // the server must deliver the manifest with the MIME type text/cache-manifest


CACHE MANIFEST                                      // the manifest must contain all files (images, fonts, ...) without any typos

scripts/my-script.js                                // if one file can't be loaded, the whole manifest will be ignored

styles/my-style.css                                 // non-listed files won't be loaded at all (not even dynamically when the browser is online)


FALLBACK:                                           // allows conditional caching: the first file is always requested from the server when

my-file.html my-offline-file.html                   // the browser is online, but will be substituted with the cached version of the second file

my-image.jpg my-image-small.jpg                     // when running offline (useful to cache smaller version of expensive assets)

online-check.js offline-check.js                    // can define a JavaScritp function that indicates if the application runs offline

/ offline-error.html                                // supports a catch-all fallback

/my-app/ offline-error.html                         // catch everything non-cached below the /my-app path


NETWORK:                                            // use NETWORK to declare non-cached assets that must always be loaded from the server

images/my-image.jpg                                 // files can be listed just like in the above sections

*                                                   // ... or use wildcards to cover a whole range of files

*.jpg

http://mydomain.com/*.jpg


SETTINGS:

prefer-online                                       // only FF: use the cache only when offline; otherwise always request files from the server



Using a cache manifest


<html manifest="/my-application.appcache">          <!-- link the manifest file to the HTML tag                          -->


applicationCache.status                             // 0 uncached, 1 idle, 2 checking, 3 downloading, 4 update ready, 5 obsolete


applicationCache.onchecking                         // looking for updates to the cached elements

applicationCache.onerror                            // check or download failed

applicationCache.onnoupdate                         // manifest has not changed (no update necessary)

applicationCache.ondownloading                      // download begins

applicationCache.onprogress                         // fired repeatedly as files are being downloaded

applicationCache.onupdateready                      // a new version has been downloaded and can be activated (call swapCache(); see below)

applicationCache.oncached                           // complete and ready for use

applicationCache.onobsolete                         // the manifest no longer exists (on the server), the offline cache has been cleared


applicationCache.update()                           // force an update (if there is something to be updated)

applicationCache.swapCache()                        // activate the newly downloaded version (will not reload the page, mixes old and new content)



Web storage


storage = window.sessionStorage                     // data is only kept as long as the session exists (requires a server session, not available for file:///)

storage = window.localStorage                       // data can be kept accross sessions and is shared among all pages on the same protocol+domain+port


storage.setItem("key", "value")                     // store data (values must be [or will be converted to] strings)

storage.getItem("key")                              // retrieve an item's value

storage.length                                      // counts the current number of entries

storage.key(index)                                  // retrieve the name of the value (i.e. the key?) at the given index

storage.removeItem("key")                           // remove a single item

storage.clear()                                     // remove all data


window.onstorage                                    // fired when changes to a storage occur (not in IE<=8; is only fired in other browser windows)

                                                    

storageEvent.domain                                 // not all properties are supported by all browsers

storageEvent.key

storageEvent.newValue

storageEvent.oldValue
