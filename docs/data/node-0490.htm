Defining variables


@myVariable: hello;

@myVariable: @myVariable + 5pt;                     // variables are constant, they can't be modified

@myVariable: 10cm + 5in;                            // ... but they can be redefined (overriding previous definitions)



Referencing variables


#myElement {

    color: @myColour;                               // reference a variable via the @-prefix as a regular value

}


.@{myClass} {                                       // use the @{myVariable} syntax for selectors

    background-@{myProperty}: #999;                 // ... or propertyNames

    background: url("@{myImageFile}");              // ... or strings like URLs

}    

@import "@{themes}/tidal-wave.less";                // ... and import statements


@@myVariable                                        // indirection: get the value of the variable whose name is stored in @myVariable


#header {

    content: ~"david@email.com";                    // use ~"..." or ~'...' to disable interpolation for a string

}



Scope and shadowing


@myVariable: red;                                   // variables can be defined globally or nested in a rule's scope


#header {

    .myClass {

        some-property: @myVariable;                 // green (variables can be used before they are defined)

        @myVariable: green;

    }

    @myVariable: brown;

    some-other-property: @myVariable;               // yellow (only the last definition in the scope is used)

    @myVariable: yellow;

}
