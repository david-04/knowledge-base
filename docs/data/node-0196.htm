API version dependencies


  • develop the changes in a [developer or full] sandbox
  • Setup => Custom Code => Apex classes (and/or triggers)
  • open the class and configure dependencies in the "Version Settings" tab



Export a change set


https://developer.salesforce.com/docs/atlas.en-us.apexcode.meta/apexcode/apex_qs_deploy.htm


  • Setup => Outbound Change Sets => New
  • add the required change set components
  • select "View/Add Dependencies" (this will list all prerequisits)
  • select all that are applicable and "Add to Change Set"
  • select "Upload" and choose the target instace (configure the sandbox in PROD: Setup => Deploy => Deployment Connections)



Import a change set


  • Environments => Change Sets => Inbound Change Sets
