Overall status


> git status -s my-file.txt                         // show a list of pending changes (-s for the short version)


> git reflog                                        // show where the HEAD moved recently (reveals recent checkouts and commits,

                                                    // can be used to detect orphaned commits that are no longer reachable)



Branch/merge history


git log --graph --color --decorate --oneline --all  // show a history of all branches



Commit history


> git log                                           // show the commit history (by default: only the current branch with its parents)

          -p                                        // include a file-based diff for each commit

          -2                                        // show only the two most recent commits (can pass any number, e.g. git log -5)


          --{since|after}={2.weeks|2016-12-24}      // show only commits after the given point of time

          --{before|until}=2016-12-24               // show only commits before the specified date


          --{author|commiter}=David                 // include only adds/commits from authors whose name contains "David"

          --grep=myComment                          // include only commits with comments containing the term "myComment"

          --all-match                               // when using --author and --grep: combine via AND (instead of the default OR)

          -SmyFunction                              // only show commits with a diff containing the term "myFunction"


          --graph                                   // include an ASCII art graph with the branch and merge history

          --decorate                                // show tags and branches as well

          --relative-date                           // show dates like "yesterday" or "2 weeks ago"

          --stat                                    // include statistics (number of files added/modified/deleted)

          --shortstat                               // include a short form of the statistics

          --abbrev-commit                           // shorten the commit's SHA-1 as much as possible

          --name-only                               // show only the names of the affected files

          --name-status                             // show the file names and their status (added/modified/deleted)

          --oneline                                 // condense each commit into one line

          --pretty={short|full|fuller}              // use a shorter or a more verbose output format

          --pretty=format:"%h - %an, %ar : %s"      // apply a custom format string

                                                    // %H .... commit hash           |  %h .... abbreviated commit hash

                                                    // %T .... tree hash             |  %t .... abbreviated tree hash

                                                    // %P .... parent hashes         |  %p .... abbreviated parent hashes

                                                    // %an ... author name           |  %cn ... commiter name

                                                    // %ae ... author email          |  %ce ... committer email

                                                    // %ad ... author date           |  %cd ... committer date

                                                    // %ar ... relative author date  |  %cr ... relative commiter date

                                                    // %s .... subject               |

          -- my-file.txt                            // optional: show only commits affecting the given file(s)


> git log myBranch --not master                     // show all commits of myBranch that are not on the master branch

> git log myBranch master..myBranch                 // same as above


> git shortlog                                      // show a condensed version of git log

> git shortlog --no-merges master --not myTag       // show all commits after myTag grouped by contributor


> git describe myBranch                             // returns something like: v1.6.2-rc1-20-g8c5b85c

                                                    //                         |          |  +-- latest commit checksum

                                                    //                         |          +-- number of commits since the tag

                                                    //                         +-- most recent tag at or before the latest commit#

                                                    // its a unqiue ID like the SHA-1 but slightly more readable


> git show 0c05aa                                   // show details of specific commit



File content


> git diff my-file.txt                              // show unstaged modifications for all files (or just the selected one)

> git diff --{staged|cached}                        // show staged but uncommited changes (--cached is a synonym for --staged)

> git diff --check                                  // check for trailing whitespace errors

> git diff master...myBranch                        // show all changes that occurred in myBranch since diverting from master


> git blame    -L 12,22 my-file.txt                 // show which commit made the most recent changes to lines 12-22

> git blame -C -L 41,53 my-file.txt                 // try to follow rename operations to trace the last modification


> git grep -n myFunction                            // search myFunction in the working directory and print the line numbers

> git grep --count myFunction                       // show only which files match and how many matches there are

> git grep -p myFunction                            // also show in which surrounding function the term was matched

> git grep --break --heading                        // group matches by file (more readable output)

> git grep myFunction --and myOtherFunction         // lines must contain both search terms


> git log -SmyFunction                              // show all commits whose change set contains the term "myFunction"

> git log "-Gmy.*Function"                          // ... supports regular expressions as well

> git log -L :myFunction:my-file.txt                // show all changes to myFunction in file my-file.txt



Bisection


> git bisect start                                  // start a bisection process

> git bisect bad                                    // mark the current revision (latest commit) as containing the bug

> git bisect good myTag                             // mark the revision with myTag as not containing the bug

...                                                 // Git will checkout a revision in the middle between myTag...HEAD

> git bisect {good|bad}                             // mark this version as containing or not containing the bug

...                                                 // Git will checkout another revision in the middle

> git bisect {good|bad}                             // ...


> git bisect reset                                  // after the culprit was identified: reset the the bisection


> git bisect start HEAD myTag                       // automated process: search a bug between myTag and HEAD

> git bisect run test-for-error.sh                  // for every intermediate commit, Git will run this script to evaluate the code

                                                    // (the script must exit with return code 0 for OK and <>0 for error)
