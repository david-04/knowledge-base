Defining themes


  • need to be placed in the App_Themes folder
  • a separate subfolder is required for each theme
  • contains CSS, images and maybe a skin file



Types of themes


StyleSheetThemes

  • provide a default look and feel
  • are applied early in the page life cycle (the <link> node is place at the beginning of the <head>)
  • can be overridden by the page


Themes

  • enforce the look and feel
  • are applied late in the page life cycle
  • can't be overridden by the page (the <link> node is added at the bottom of the <head>)
  • exception: set EnableTheming=false for a control to disable theming



Applying themes


Globally via web.config:


<configuration>

  <system.web>

    <pages theme="MyTheme" styleSheetTheme="MyTheme">

    </pages>


On page level via declaration:


<%@ Page ... Theme="MyTheme" StyleSheetTheme="MyTheme" %>


On page level via code:


private void Page_PreInit(object sender, EventArgs e)

{

    string theme = "MyTheme";

    string folder = Server.MapPath("~/App_Themes/" + theme);

    if (System.IO.Directory.Exists(folder))

    {

        Page.Theme = theme;

    }

}



Disabling themes


<asp:Button ID="Submit" runat="server" Text="Submit" EnableTheming="False" />
