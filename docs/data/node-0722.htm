Processing order


The parser traverses the whole XML document:

  • the parent node is processed first before recursing into the children
  • the parser does not automatically process attributes (they are ignored)
  • user-defined template rules are matched via an XPath expression
  • if no user-define rule matches, the built-in default rule is applied


If there is no matching template rule:

  • if the process node has text content, it is copied to the output
  • attributes, comments, processing instructions and namespaces are ignored


If there is a matching template rule:

  • the template is applied and the automatic recursion stops
  • use <xsl:apply-templates/> within a template rule to resume automatic recursion
  • use <xsl:apply-templates select="@..."/> to recurse into attributes



Syntax


<?xml version="1.0" encoding="utf-8"?>              <!-- the XSLT has a regular XML prologue                             -->

<xsl:stylesheet                                     <!-- can use the synonym xsl:transform instead of xsl:stylesheet     -->

  version="1.0"

  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

 

  <xsl:output encoding="utf-8"                      <!-- optional: configure the output format                           -->

              method="xml"                          <!-- xml, html or text                                               -->

              indent="yes"                          <!-- pretty-print the result (or not)                                -->

              ... />                                <!-- other settings are available as well                            -->


  <xsl:template match="person">                     <!-- this rule applies to every <person> anywhere in the document    -->

    <b>Person:</b>                                  <!-- the output can contain HTML/XML (but must be well-formed)       -->

    <xsl:apply-templates select="name"/>            <!-- process all <name> elements anywhere in the current <person>    -->

    (<xsl:value-of select="@age"/>)                 <!-- insert an attribute value from the currently processed <person> -->

  </xsl:template>


  <xsl:template match="name">                       <!-- matches <name> anywhere in the document (not only in <person>)  -->

    <xsl:value-of select="first_name"/>             <!-- retrieve the text content of child nodes                        -->

    <xsl:value-of select="last_name"/>

  </xsl:template>


</xsl:stylesheet>
