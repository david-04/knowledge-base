Creating patches


> git format-patch -M origin/master                        // create patch files for all commits on the master branch



Applying patches


> git am my.patch                                   // apply a patch that was generated with git format-patch

> git am -3 my.patch                                // apply a 3-ways-diff (the patch branch's starting point must exist within the local repository)


> git apply my.patch                                // apply a patch that was generate with git diff or the UNIX built-in diff command

> git apply --check my.patch                        // dry-run, see if the patch would merge cleanly



Mailing patches


[imap]                                              // define IMAP settings in gitconfig

  folder    = "[Gmail]/Drafts"

  host      = imaps://imap.gmail.com

  user      = user@gmail.com

  pass      = p4ssw0rd

  port      = 993

  sslverify = false


[sendemail]                                         // define SMTP settings in gitconfig

  smtpencryption = tls

  smtpserver     = smtp.gmail.com

  smtpuser       = user@gmail.com

  smtpserverport = 587


> cat *.patch | git imap-send                       // create emails with the patches in the drafts folder of the IMAP account

> git send-email *.patch                            // send the patches via SMTP (will prompt for the recipients' addresses)

> git request-pull origin/master myBranch           // create a sample email message body for a pull request



Bundles


> git bundle create my.bundle HEAD master           // create a bundle file with all commits between master and HEAD

> git clone my.bundle ./myRepository                // clone the bundle into a new repository

> git bundle verify my.bundle
