Numeric data types


INTEGER / INT                                       -- -2,147,483,648 ... +2,147,483,647

SMALLINT                                            -- -32,768 ... 32,767

TINYINT                                             -- 0 ... 255

BIGINT                                              -- -2^63 ... 2^63-1

DECIMAL(p,[s]) / DEC / NUMERIC                      -- fixed point decimals, p=total number of digits, s=digits after the decimal point

REAL                                                -- floating-point +/-3.40E38 ... +/-1.18e-38

FLOAT(n)                                            -- floating-point, n=number of bits used to store the mantissa

MONEY                                               -- 8-byte decimal with four decimal digits

SMALLMONEY                                          -- 4-byte decimal with four decimal digits



String data types


CHAR(n) / CHARACTER                                 -- fixed-length string

VARCHAR(n)                                          -- variable-length single-byte characters

NCHAR(n)                                            -- fixed-length unicode string (2 bytes per character)

NVARCHAR(n)                                         -- variable-length unicode string (2 bytes per character)


XML                                                 -- an XML document



Temporal data types


DATETIME                                            -- 01/01/1753 - 31/12/9999

SMALLDATETIME                                       -- 01/01/1900 - 06/06/2079

DATE                                                -- 01/01/0001 - 31/12/9999

TIME                                                -- accuracy of 100 nanoseconds

DATETIME2                                           -- high precicion date and time (not DST-aware)

DATETIMEOFFSET                                      -- includes a time zone offset


'mmm dd yyyy', e.g., 'Jan 10 1993'                  -- default date format

'28/5/1959' (with SET DATEFORMAT dmy)



Other data types


BINARY(n)                                           -- fixed length bit string

VARBINARY(n)                                        -- variable length bit string

VARCHAR(max)                                        -- variable length large binary objects (up to 2GB)

NVARCHAR(max)

VARBINARY(max)                                      -- add the FILESTREAM attribute to save this column as separate files on the file system

UNIQUEIDENTIFIER                                    -- similar to the global unique identifier (GUID), initialised via NEWID or NEWSEQUENTIALID

                                                    -- mark as ROWGUIDCOL if the column should be used as the primary ROWID

SQL_VARIANT                                         -- can store any data type except TIMESTAMP, use SQL_VARIANT_PROPERTY to identify the data type

HIERARCHYID                                         -- hierarchy of objects

TIMESTAMP / ROWVERSION                              -- not the current time ... a database-wide, unique sequence

SPARSE                                              -- additional attribute: NULL values consume no disk space at all, but non-NULL values require 2-4 additional bytes
