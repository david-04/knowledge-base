Defining complex types


<xs:element name="myElement">                       <!-- embed the type definition into the element declaration        -->

  <xs:complexType>

    ...

  </xs:complexType>

</xs:element>





<xs:complexType name="myType">                      <!-- define a re-usable type with a dedicated name                 -->

  ...

</xs:complexType>


<xs:element name="myElement" type="myType" />       <!-- ... and reference the type in the type definition             -->





<xs:group name="myGroup">                           <!-- define a re-usable all, choice or sequence group              -->

  <xs:{all|choice|sequence}>

    ...

  </xs:{all|choice|sequence}>

</xs:group>


<xs:element name="myElement">

  <xs:complexType>

    <xs:group ref="myGroup"/>                       <!-- ... and reference the group in the type definition            -->

  </xs:complexType>

</xs:element>



Sequences


<xs:complexType name      = "myType"       >        <!-- sequence: all children are mandatory and must occur in        -->

  <xs:sequence>                                     <!-- the same order as they are specified here                     -->

    <xs:element name      = "myElement1"

                type      = "xs:string"   />

    <xs:element name      = "myElement2"

                type      = "xs:string"

                minOccurs = "0"                     <!-- elements can be optional and/or repetable                     -->

                maxOccurs = "unbounded"   />

  </xs:sequence>

</xs:complexType>





<xs:complexType name      = "myType"       >        <!-- unsorted sequence: all children must occur excactly once      -->

  <xs:all>                                          <!-- but can be in any order                                       -->

    <xs:element name      = "myElement1"

                type      = "xs:string"   />

    <xs:element name      = "myElement2"

                type      = "xs:string"

                minOccurs = "0"           />        <!-- elements can be optional but not repeatable                   -->

  </xs:all>

</xs:complexType>





<xs:complexType name      = "myType"       >        <!-- choice: exactly one child must be present                     -->

  <xs:choice>

    <xs:element name      = "myElement1"

                type      = "xs:string"   />

    <xs:element name      = "myElement2"

                type      = "xs:string"

                minOccurs = "0"                     <!-- if a child is optional, the parent can be empty               -->

                maxOccurs = "unbounded"   />        <!-- elements can be repeatable                                    -->

  </xs:choice>

</xs:complexType>



Nested documents


<xs:any namespace       = "http://my-domain.com"    <!-- allow any element from the (optional) namespace               -->

        minOccurs       = "0"                       <!-- minOccurs and maxOccurs is applicable just like with every    -->

        maxOccurs       = "unbounded"               <!-- regular <xs:element> declaration                              -->

        processContents = "skip"             />     <!-- skip ..... do not validate the element and its content        -->

                                                    <!-- lax ...... validate known elements and ignore everything else -->

                                                    <!-- strict ... validate everything                                -->
