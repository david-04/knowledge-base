Initialisation


> git init                                          // initialise the current folder as a repository (won't add/commit any files)

> git init --bare --shared                          // initialise the current folder as a bare repository (used by everyone to push to

                                                    // and pull from; doesn't have any checked out local working copy; the folder name

                                                    // should end with the suffix ".git" by convention)



Cloning


> git clone https://github.com/user/project         // clone an existing repository (creates a folder named "project")

> git clone https://github.com/user/project work    // clone the repository in a folder named "work"


> git clone https://github.com/user/project         // clone from a HTTP server

> git clone ssh://user@server/project.git           // clone from a SSH server (the ssh:// is optional and can be omitted)

> git clone /opt/git/project.git                    // clone from the file system and create links instead of copying all files

> git clone file:///opt/git/project.git             // clone from the file system but copy all files instead of creating links


> git clone --bare https://... my-repository.git    // clone the specified repository and create a bare copy of it (the target

                                                    // folder name should end with the suffix ".git" by convention)


> git clone --recursive  ...                        // clone all embedded submodules as well



Managing remotes


> git remote -v                                     // show the remote repositories that are linked to the current (local) one

> git remote show {origin|myAlias}                  // show details about a remote repository (e.g. new branches)


> git remote add myAlias https://...                // link an additional respository (known locally under the alias "myAlias")

> git remote rename myAlias myNewAlias              // rename a remote repository alias

> git remote rm myAlias                             // remove a selected remote repository



Uploading changes


> git push {origin|myAlias} {master|myBranch}       // push a branch to a remote repository (won't push any other branches/tags)

> git push {origin|myAlias} myBranch:davidsBranch   // push a branch but rename it on the remote repository

> git push {origin|myAlias} myTag                   // push a tag

> git push {origin|myAlias} --tags                  // push all new tags


> git push {origin|myAlias} --delete myBranch       // delete a branch from a remote repository



Downloading changes


> git fetch {origin|myAlias}                        // pull the latest commits from the specified (or all) repositories

                                                    // this won't merge any changes but create new (duplicate) branches


> git log --no-merges myBranch..origin/myBranch     // show all newly fetched commits in myBranch that need to be merged

> git merge {origin|myAlias}/myBranch               // merge the fetched myBranch into the current local branch


> git pull {origin|myAlias} myBranch                // do a git fetch + git merge (works only for tracking branches)


> git pull                                          // do a git fetch + git merge into the current local branch
