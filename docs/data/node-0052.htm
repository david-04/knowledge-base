Transactions


BEGIN TRANSACTION my_transaction WITH MARK 'desc';  -- name is optional (only for outermoste transactions), the MARK is a note for the transaction log


COMMIT/ROLLBACK WORK;                               -- WORK is optional, refers to the current (innermost) transaction

COMMIT/ROLLBACK my_transaction;                     -- commit or rollback a named transaction


SAVE TRANSACTION my_save_point;                     -- like a Windows restore point

ROLLBACK TRANSACTION my_save_point;                 -- rollback to a previous restore point



Locks


SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;   -- ignore all locks an read data even if it has not (yet) been committed

SET TRANSACTION ISOLATION LEVEL READ COMMITTED;     -- place a shared lock during the read operation, read only commited data, release the lock straight afterwards

SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;    -- place a shared lock and prevent other transactions from manipulating the data ... the lock is closed at the end of the transaction

SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;       -- exclusive table lock ... also prevents others from inserting new records


SET TRANSACTION ISOLATION LEVEL SNAPSHOT;           -- used for optimistic concurrency only
